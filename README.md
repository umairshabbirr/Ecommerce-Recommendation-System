# E-Commerce Recommendation System (C++)

A console-based e-commerce system in C++ that recommends products to users based on collaborative filtering — built using object-oriented design and a custom linked-list data structure.

![C++](https://img.shields.io/badge/C++-OOP-blue?logo=cplusplus)
![Status](https://img.shields.io/badge/Status-Active-brightgreen)

## Overview

Most beginner C++ course projects stop at basic CRUD operations. This one goes further: it implements a simplified **collaborative filtering recommendation engine** — the same core idea behind "customers who bought this also bought..." systems used by real e-commerce platforms — built entirely from scratch using OOP principles and a custom linked list, without any external libraries.

Users can sign up, log in, browse products, make purchases, and receive product recommendations generated by comparing their purchase history against other users' behavior.

---

## Key Features

- User signup & login system
- Product catalog with polymorphic product classes (extendable to other categories beyond men's clothing)
- Custom-built **linked list** to track each user's purchase history (no STL containers used for this part — implemented manually to demonstrate core DSA understanding)
- **Collaborative filtering recommendation engine** — finds the most "similar" user based on shared purchase history, then recommends products that user bought but the current user hasn't
- Persistent session loop — users can log out and switch accounts without restarting the program

---

## How the Recommendation Logic Works

1. Every user's purchases are stored in their own linked list (`PurchaseHistory`)
2. When a recommendation is needed, the system calculates a **similarity score** between the current user and every other user — counting how many products they've both purchased
3. The user with the highest similarity score is treated as the "closest match"
4. Products that the matched user purchased — but the current user hasn't — are recommended

This is a simplified, from-scratch implementation of **user-based collaborative filtering**, the same concept behind recommendation systems used in production by platforms like Amazon.

---

## Tech Stack

- **Language:** C++
- **Core Concepts Used:** Object-Oriented Programming (inheritance, polymorphism, abstract classes), custom linked list implementation, vectors, pointers

No external libraries — built using the C++ Standard Library's `<iostream>` and `<vector>` only, with the core data structure (purchase history) implemented manually.

## Getting Started

**Requirements:** Any C++ compiler (g++, MinGW, or an IDE like Code::Blocks / Visual Studio)

```bash
# 1. Clone the repository
git clone https://github.com/umairshabbirr/Ecommerce-Recommendation-System.git
cd Ecommerce-Recommendation-System

# 2. Compile
g++ recommendation_engine.cpp -o recommendation_engine

# 3. Run
./recommendation_engine
```

Then:
1. Sign up with a new username and password (or explore with the pre-seeded demo users)
2. Browse the product catalog and purchase items
3. After each purchase, the system generates personalized recommendations based on similar users
